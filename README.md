## Flujo de trabajo

Este proyecto utiliza el modelo **GitFlow** para una colaboración estructurada, con ramas:

- `main`: versión estable
- `develop`: rama base para desarrollo
- `feature/*`: nuevas funcionalidades
- `release/*`: preparación de despliegue
- `hotfix/*`: correcciones urgentes
